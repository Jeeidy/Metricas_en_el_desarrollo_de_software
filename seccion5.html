Métricas para Pruebas

La importancia de las métricas para las pruebas de software, señalando que la mayoría de las métricas se centran en el proceso de pruebas en lugar de en las características técnicas de las pruebas en sí. Propone el uso de métricas basadas en la función para predecir el esfuerzo global de las pruebas y correlacionar características a nivel de proyecto con el número de pruebas producidas. También sugiere el uso de métricas de diseño de alto nivel, como la complejidad ciclomática, para evaluar la dificultad asociada con la prueba de integración. Además, ofrece fórmulas para estimar el esfuerzo de las pruebas y asignar el esfuerzo de pruebas a diferentes módulos del sistema. Finalmente, sugiere medidas como la amplitud y profundidad de las pruebas, así como el uso de perfiles de fallos, para priorizar y categorizar los errores descubiertos durante las pruebas.

Métricas de Mantenimiento

Las métricas están específicamente diseñadas para actividades de mantenimiento de software. Se destaca el índice de madurez del software (IMS) propuesto en el estándar IEEE 982.1-1988, el cual indica la estabilidad de un producto de software en función de los cambios que ocurren en cada versión del mismo. El IMS se calcula considerando el número de módulos en la versión actual, los módulos que se han cambiado, añadido o borrado con respecto a la versión anterior. A medida que el IMS se acerca a 1.0, el producto se estabiliza. Además, se sugiere que el IMS puede utilizarse como métrica para planificar actividades de mantenimiento, y que el tiempo medio para producir una versión del producto puede correlacionarse con el IMS para desarrollar modelos empíricos de mantenimiento.
